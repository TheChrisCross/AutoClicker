# Security Policy

## Supported Versions

Security fixes are considered for the latest published release and the current
`main` branch. Older releases are not actively supported; users should upgrade
to the latest release after a fix is published.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Current `main` branch | Best effort |
| Older releases | No |

## Report a Vulnerability

Do not open a public issue for a suspected vulnerability.

Email the maintainer privately at ezharddscope@gmail.com with the subject
`[AutoClicker Security]` and include, when available:

- The affected release version or commit hash
- A clear description of the vulnerability and its potential impact
- Reproduction steps or a minimal proof of concept
- Relevant Windows version and configuration details
- Any suggested mitigation or fix

Remove passwords, tokens, personal data, and unrelated sensitive information
from the report. The maintainer will review the report, may request additional
details, and will coordinate disclosure when a fix or mitigation is available.
Please allow a reasonable amount of time for investigation before publishing
details.

Reports about general bugs, feature requests, or usage questions should use the
repository's public issue templates instead.
